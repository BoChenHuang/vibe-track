# VibeTrack 四週開發計畫

**總時數：** 64 小時（每週 16 小時）

---

## Week 1｜後端骨架

**目標：** API 能跑起來，文字輸入能通

| 任務 | 時數 |
|------|------|
| 專案初始化、資料夾結構、Git 設定 | 1h |
| 設計 API 介面（endpoint、request/response 格式） | 2h |
| 文字輸入 → Claude API 整合，回傳氛圍描述 | 4h |
| Claude 回傳結果 → Spotify API 搜尋歌曲 | 4h |
| 組合最終回傳格式（8 首歌 + 推薦理由） | 3h |
| 基本錯誤處理與輸入驗證 | 2h |

**里程碑：** 用 Postman 打文字輸入，能拿到歌單

---

## Week 2｜圖片輸入 + 防護機制

**目標：** 圖片輸入上線，費用保護到位

| 任務 | 時數 |
|------|------|
| 圖片上傳處理（格式驗證、大小限制） | 3h |
| 圖片 → Claude Vision API 整合 | 4h |
| Rate Limiting 實作（Redis 計數器） | 3h |
| Redis 快取實作（hash 輸入當 key） | 4h |
| 後台設定 Claude API 費用上限 | 1h |
| 資料庫設計（PostgreSQL + Prisma schema） | 2h |
| 查詢紀錄寫入與查詢 API | 2h |
| 測試與 bug 修正 | 1h |

**里程碑：** 圖片和文字都能用，防護機制上線，查詢紀錄可存取

---

## Week 3｜前端 Demo 頁面

**目標：** 有可以展示的畫面

| 任務 | 時數 |
|------|------|
| 前端環境建立（Vite + Tailwind） | 1h |
| 文字輸入介面 | 2h |
| 圖片上傳介面 | 2h |
| 呼叫後端 API、顯示歌單列表 | 4h |
| 30 秒試聽播放器 | 3h |
| Spotify 連結跳轉 | 1h |
| 整體 UI 調整、RWD | 3h |

**里程碑：** 完整 demo 流程可以跑，能錄影

---

## Week 4｜部署 + 收尾

**目標：** 上線、面試能展示

| 任務 | 時數 |
|------|------|
| 後端部署（Railway 或 Fly.io） | 3h |
| 前端部署（Vercel） | 2h |
| 環境變數與設定整理 | 1h |
| API 文件撰寫（Swagger 或 README） | 4h |
| README 撰寫（專案說明、架構圖、Future Work） | 3h |
| 全流程測試、bug 修正 | 3h |

**里程碑：** 有公開網址，README 完整，可以放上履歷

---
